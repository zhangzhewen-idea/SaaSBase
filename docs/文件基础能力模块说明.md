# 文件基础能力模块说明

日期：2026-07-13

## 1. 模块用途

文件基础能力模块用于租户后台的文件上传、查询、预览、下载和删除。

当前实现只支持本地存储，接口对外不暴露 `objectKey`、本地路径或其他存储细节。

## 2. 接口列表

基础路径：

```text
/api/v1/admin/files
```

接口如下：

```text
POST   /api/v1/admin/files
GET    /api/v1/admin/files/{id}
GET    /api/v1/admin/files
GET    /api/v1/admin/files/{id}/content?disposition=inline|attachment
DELETE /api/v1/admin/files/{id}
```

## 3. 上传

请求方式：

```text
multipart/form-data
```

文件字段名固定为 `file`。

上传成功后返回文件元数据，包含：

- `id`
- `tenantId`
- `originalFilename`
- `contentType`
- `extension`
- `size`
- `status`
- `createdAt`
- `createdBy`

不返回：

- `objectKey`
- 存储类型
- 本地文件路径

## 4. 详情与列表

详情接口仅返回当前租户下可见的已上传文件。

列表接口支持筛选参数：

- `filename`：原文件名模糊查询
- `contentType`：MIME 类型精确匹配
- `uploadedFrom`：上传时间下界
- `uploadedTo`：上传时间上界
- `pageNo`：页码，默认 `1`
- `pageSize`：每页条数，默认 `10`

默认排序为：

```text
created_at DESC, id DESC
```

## 5. 预览与下载

内容接口支持：

```text
disposition=inline
disposition=attachment
```

规则如下：

- 默认值是 `attachment`
- `inline` 只对安全白名单中的 MIME 类型生效
- 响应头包含 `Content-Type`、`Content-Length`、`Content-Disposition` 和 `X-Content-Type-Options: nosniff`

## 6. 删除

删除成功返回 `204 No Content`。

重复删除、越权删除和不存在文件统一按“文件不存在”处理。

## 7. 权限点

当前接口使用以下权限点：

- `tenant:file:write`
- `tenant:file:read`
- `tenant:file:delete`

## 8. 配置

配置前缀为 `saasbase.file`：

```yaml
saasbase:
  file:
    root-path: ./data/files
    max-size: 20MB
    allowed-extensions: [pdf, png, jpg, jpeg, docx, xlsx]
    allowed-content-types:
      - application/pdf
      - image/png
      - image/jpeg
      - application/vnd.openxmlformats-officedocument.wordprocessingml.document
      - application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
    inline-content-types: [application/pdf, image/png, image/jpeg]
```

## 9. 调用约定

- 所有文件接口都要求登录态。
- 租户由认证上下文决定，不接受请求参数传入。
- 外部调用不要依赖 `objectKey`。
- 不要假设同名文件会覆盖，重复上传会生成新的文件记录。
