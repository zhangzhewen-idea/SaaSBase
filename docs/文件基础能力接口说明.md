# 文件基础能力接口说明

日期：2026-07-13

本文用于前端联调和接口调用。

## 1. 统一响应

接口统一使用项目公共响应结构：

```json
{
  "success": true,
  "code": "OK",
  "message": "OK",
  "data": {}
}
```

失败时：

```json
{
  "success": false,
  "code": "FILE_NOT_FOUND",
  "message": "文件不存在",
  "data": null
}
```

## 2. 上传文件

```text
POST /api/v1/admin/files
```

请求类型：

```text
multipart/form-data
```

字段：

- `file`：文件内容，必填

示例：

```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "file=@report.pdf" \
  http://localhost:8080/api/v1/admin/files
```

响应示例：

```json
{
  "success": true,
  "code": "OK",
  "message": "OK",
  "data": {
    "id": 7,
    "tenantId": 2001,
    "originalFilename": "report.pdf",
    "contentType": "application/pdf",
    "extension": "pdf",
    "size": 12345,
    "status": "AVAILABLE",
    "createdAt": "2026-07-13T08:00:00Z",
    "createdBy": 3001
  }
}
```

## 3. 文件详情

```text
GET /api/v1/admin/files/{id}
```

示例：

```bash
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/admin/files/7
```

## 4. 文件列表

```text
GET /api/v1/admin/files
```

查询参数：

- `filename`
- `contentType`
- `uploadedFrom`
- `uploadedTo`
- `pageNo`
- `pageSize`

示例：

```bash
curl -H "Authorization: Bearer <token>" \
  "http://localhost:8080/api/v1/admin/files?filename=report&pageNo=1&pageSize=10"
```

响应中的 `data` 为分页结构：

```json
{
  "items": [],
  "total": 0,
  "pageNo": 1,
  "pageSize": 10
}
```

## 5. 读取文件内容

```text
GET /api/v1/admin/files/{id}/content?disposition=inline|attachment
```

默认值：

```text
disposition=attachment
```

说明：

- `attachment`：浏览器按下载处理
- `inline`：仅对白名单中的 MIME 类型返回内联展示

响应头：

- `Content-Type`
- `Content-Length`
- `Content-Disposition`
- `X-Content-Type-Options: nosniff`

示例：

```bash
curl -L -H "Authorization: Bearer <token>" \
  "http://localhost:8080/api/v1/admin/files/7/content?disposition=inline" \
  -o report.pdf
```

## 6. 删除文件

```text
DELETE /api/v1/admin/files/{id}
```

成功返回 `204 No Content`。

## 7. 错误码

文件模块常见错误码如下：

| code | 含义 | HTTP 状态 |
| --- | --- | --- |
| `FILE_INVALID` | 文件无效 | 400 |
| `FILE_SIZE_EXCEEDED` | 文件大小超过限制 | 413 |
| `FILE_TYPE_NOT_ALLOWED` | 不支持的文件类型 | 415 |
| `FILE_NOT_FOUND` | 文件不存在 | 404 |
| `FILE_STATE_INVALID` | 文件状态无效 | 409 |
| `FILE_STORAGE_FAILED` | 文件存储失败 | 500 |
| `FILE_READ_FAILED` | 文件读取失败 | 500 |
| `FILE_DELETE_FAILED` | 文件删除失败 | 500 |

## 8. 调用注意事项

- 所有接口都需要登录态。
- 租户由后端认证上下文决定，前端不要传租户 ID。
- 列表接口默认分页参数为 `pageNo=1`、`pageSize=10`。
- 对外页面不要依赖 `objectKey`，该字段不会出现在返回值中。
- 重复上传不会覆盖旧文件，会生成新的文件记录。
