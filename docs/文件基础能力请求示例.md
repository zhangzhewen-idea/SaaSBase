# 文件基础能力请求示例

日期：2026-07-13

以下示例可用于本地联调和后端自测。

## 1. 上传文件

```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  -F "file=@./examples/report.pdf" \
  http://localhost:8080/api/v1/admin/files
```

响应示例：

```json
{
  "success": true,
  "code": "OK",
  "message": "OK",
  "data": {
    "id": 7,
    "tenantId": 2001,
    "originalFilename": "report.pdf",
    "contentType": "application/pdf",
    "extension": "pdf",
    "size": 12345,
    "status": "AVAILABLE",
    "createdAt": "2026-07-13T08:00:00Z",
    "createdBy": 3001
  }
}
```

## 2. 查询详情

```bash
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/admin/files/7
```

## 3. 查询列表

```bash
curl -H "Authorization: Bearer <token>" \
  "http://localhost:8080/api/v1/admin/files?filename=report&contentType=application/pdf&pageNo=1&pageSize=10"
```

## 4. 预览文件

```bash
curl -L -H "Authorization: Bearer <token>" \
  "http://localhost:8080/api/v1/admin/files/7/content?disposition=inline" \
  -o report.pdf
```

## 5. 下载文件

```bash
curl -L -H "Authorization: Bearer <token>" \
  "http://localhost:8080/api/v1/admin/files/7/content?disposition=attachment" \
  -o report.pdf
```

## 6. 删除文件

```bash
curl -X DELETE \
  -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/admin/files/7
```

成功时返回 `204 No Content`。

## 7. 常见错误

```json
{
  "success": false,
  "code": "FILE_NOT_FOUND",
  "message": "文件不存在",
  "data": null
}
```

```json
{
  "success": false,
  "code": "FILE_SIZE_EXCEEDED",
  "message": "文件大小超过限制",
  "data": null
}
```

## 8. 自测建议

- 先上传 PDF，再测试 `inline`
- 再上传 PNG/JPG，确认浏览器可内联预览
- 上传一个超限文件，确认返回 `413`
- 删除后再次查询详情，确认返回 `404`
